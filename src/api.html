Introduction

Welcome to the Discord Webhook API documentation. Learn how to send rich embeds to Discord channels using webhooks.

Quick Start

Get up and running with webhook embeds in minutes

Secure

Built-in rate limiting and validation

Flexible

Support for complex embed structures

Quick Example

// Simple webhook execution const webhook = { username: "Embed Creator", avatar_url: "https://example.com/avatar.png", embeds: [{ title: "Hello World!", description: "This is my first webhook embed", color: 0x5865F2, timestamp: new Date().toISOString() }] }; fetch('YOUR_WEBHOOK_URL', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(webhook) });

Authentication

Discord webhooks don't require traditional authentication. Instead, they use unique URLs that contain the webhook ID and token.

Webhook URL Structure

Webhook URLs follow this pattern:

https://discord.com/api/webhooks/{webhook.id}/{webhook.token}

Security Notice

Keep your webhook URLs private! Anyone with the URL can send messages to your channel.

Rate Limits

Discord enforces rate limits to ensure fair usage and prevent spam.

Endpoint Rate Limit Bucket Reset
Execute Webhook 5 requests per 2 seconds Per webhook 2 seconds
Get/Modify Webhook 10 requests per 10 seconds Per webhook 10 seconds
Global Rate Limit 50 requests per second Per IP 1 second

Handling Rate Limits

async function executeWebhook(url, data) { const response = await fetch(url, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(data) }); if (response.status === 429) { const retryAfter = response.headers.get('Retry-After'); console.log(`Rate limited! Retry after ${retryAfter}ms`); await new Promise(resolve => setTimeout(resolve, retryAfter)); return executeWebhook(url, data); // Retry } return response; }

Execute Webhook

Send a message to a Discord channel through a webhook.

POST /webhooks/{webhook.id}/{webhook.token}

Best Practices

Follow these guidelines to create effective and user-friendly webhook embeds.

Design

  • Use consistent colors across your embeds
  • Keep titles concise and descriptive
  • Use fields for structured data
  • Include timestamps for time-sensitive content

Performance

  • Respect rate limits to avoid throttling
  • Batch multiple embeds when possible
  • Cache webhook URLs securely
  • Implement retry logic for failed requests

Security

  • Never expose webhook URLs publicly
  • Validate all input data
  • Use HTTPS for all requests
  • Rotate webhook URLs periodically

User Experience

  • Provide clear and actionable content
  • Use mentions sparingly
  • Include relevant links and resources
  • Test embeds before deployment
